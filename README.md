# 🎵 Spotify Product Dissection & PostgreSQL Schema Design

A comprehensive PostgreSQL database design and SQL analytics project based on Spotify. This project explores Spotify's product architecture, database schema design, relational modeling, and business insights using PostgreSQL. This project was given by AlmaBetter.

> 📚 This project was completed as part of the **Product Dissection & PostgreSQL Schema Design** assignment provided by **AlmaBetter**.

---

## 📌 Project Overview

Spotify is one of the world's leading music streaming platforms, serving millions of users across the globe. This project focuses on understanding Spotify's core product features and designing a normalized PostgreSQL database schema capable of supporting its major functionalities.

The project includes:

- Product Dissection
- Company Research
- Real-World Problem Analysis
- PostgreSQL Database Schema Design
- Entity Relationship (ER) Diagram
- SQL Data Analysis
- Business Insight Queries
- Advanced PostgreSQL Queries
- Database Optimization using Constraints & Indexes

---

## 🎯 Objectives

- Understand Spotify's business model and platform architecture.
- Design a normalized PostgreSQL database.
- Implement relationships using Primary Keys and Foreign Keys.
- Perform Exploratory Data Analysis (EDA).
- Generate business insights using SQL.
- Apply advanced PostgreSQL concepts including Window Functions and Common Table Expressions (CTEs).

# 🗄️ Database Design

The database schema is designed using Third Normal Form (3NF) to reduce redundancy and maintain data integrity.

### Main Entities

- Users
- Artists
- Albums
- Genres
- Tracks
- Playlists
- Playlist_Tracks
- Listening_History
- Subscription_Plans
- Subscriptions
- Recommendations
